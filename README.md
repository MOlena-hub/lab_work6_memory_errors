## Практична робота №6: Дослідження переповнення стеку та купи

**Варіант №5**: Реалізувати окремі приклади переповнення стеку та купи й порівняти результати виявлення помилок через ASan, Valgrind і обмеження ulimit.

## Опис проєкту
Цей проєкт реалізує низькорівневі програми на мові C для дослідження вразливостей пам'яті в ОС Linux. Програми імітують критичні помилки керування пам'яттю, що дозволяє порівняти методи їх діагностики.

Проєкт демонструє:
1. **Переповнення стеку (Stack Overflow)**: реалізоване через нескінченну рекурсію, що призводить до вичерпання виділеного стеку процесу.
2. **Переповнення купи (Heap Overflow)**: реалізоване через запис даних за межі динамічно виділеної пам'яті за допомогою `memset`.
3. **Використання інструментів аналізу**: AddressSanitizer (ASan) та Valgrind для виявлення порушень адресації.

## Логіка програм
### Stack Overflow (`stack_overflow.c`)
- Програма викликає рекурсивну функцію, яка створює масив на стеку при кожному виклику.
- Оскільки умова виходу відсутня, стек процесу заповнюється до досягнення системного ліміту розміру стеку.

### Heap Overflow (`heap_overflow.c`)
- **malloc**: виділення 16 байт пам'яті у купі.
- **memset**: спроба записати 64 байти в 16-байтний буфер.
- **Detection**: програма демонструє, як різні інструменти реагують на вихід за межі виділеного блоку.

## Дослідження помилок пам'яті
Проведено порівняльний аналіз трьох методів контролю пам'яті:

| Інструмент | Результат (Стек) | Результат (Купа) | Особливості |
|-----------|-----------------|------------------|-------------|
| **ASan** | **Виявляє** | **Виявляє** | Зупиняє програму з детальним звітом (redzones). |
| **Valgrind** | Не виявляє | **Виявляє** | Показує деталі `Invalid write`, але дозволяє програмі завершитись. |
| **ulimit -s** | **Обмежує** | Не впливає | Просто генерує `Segmentation fault` без діагностики. |

## Як запустити
1. **Збірка проєкту (всіх версій):**
   ```bash
   make
   ```
2. **Тестування через AddressSanitizer:**
   ```bash
   ./heap_asan
   ./stack_asan
   ```
3. **Аналіз через Valgrind:**
   ```bash
   valgrind ./heap_test
   ```
4. **Перевірка системних лімітів**
   ```bash
   ulimit -s 100
   ./stack_test
   ```
